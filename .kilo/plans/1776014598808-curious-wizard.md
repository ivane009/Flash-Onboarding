## Bug Fix Required
The current `src/components/Landing.jsx` has a bug: the `useEffect` for loading CSS is called at the top level outside the component function. This violates React's rules of hooks.

**Fix:** Move the CSS loading `useEffect` inside the `Landing` component function, after the `useState` declarations.

```jsx
// WRONG (current):
useEffect(() => { ... });  // Top level - ERROR!
export default function Landing() { ... }

// CORRECT (fixed):
export default function Landing() {
  useEffect(() => { ... });  // Inside component - OK
  ...
}
```

This fix needs to be applied before the app can run.

## Bug Fix: Card Refresh on Click (crear-cuenta.html)

**Problem:** Touching the card causes the page to refresh/render multiple times.

**Root Cause:** The `document.addEventListener('click', handleClickOutside)` is called inside `render()`, which means a NEW event listener is added every time `render()` is called. This creates multiple listeners that can cause conflicting behavior.

**Solution:** Move the event listener outside of render, adding it only once.

**Implementation:**
1. In `crear-cuenta.html`:
   - Remove `document.addEventListener('click', handleClickOutside)` from inside `render()`
   - Add a flag or use `{ once: true }` to ensure listener is only added once
   - Or better: add the listener once outside of the render function

2. Same fix for `iniciar-sesion.html`

**Code change for crear-cuenta.html:**
```javascript
// BEFORE (in render):
document.addEventListener('click', handleClickOutside);

// AFTER (outside render, only once):
if (!window.clickHandlerInitialized) {
  window.clickHandlerInitialized = true;
  document.addEventListener('click', handleClickOutside);
}
```

Or simply remove the listener call from render entirely and call it once after the initial render.

## Bug Fix: Inputs and Dropdowns Cause Page Refresh

### Problema identificado

1. **`updateField` llama `render()` en cada tecla** - Cada vez que el usuario escribe, se re-renderiza todo el HTML. Esto causa que la página "flashee" o "refreshee" constantemente.

2. **`setTimeout` con `closeDropdown` dentro de `render()`** - Cada vez que se llama `render()`, se añade un nuevo `setTimeout` con un nuevo event listener. Esto acumula múltiples listeners.

3. **Dropdown de idioma** - Cuando se hace click, llama `render()` que causa el mismo problema de re-renders.

### Solución

1. **Eliminar `render()` de `updateField`**
   - Solo actualizar el estado, no re-renderizar
   - Para mostrar errores, usar validación en `onblur` o `onchange`
   - Para el submit, sí llamar `render()` una vez al validar

2. **Eliminar el `setTimeout` de `render()` completamente**
   - Añadir UN SOLO event listener fuera de `render()`
   - Usar event delegation para cerrar dropdowns

3. **Simplificar toggles de dropdown**
   - `toggleLangDropdown()` y `toggleCountryDropdown()` no deben llamar `render()`
   - En su lugar, usar CSS classes para mostrar/ocultar

### Cambios específicos

```javascript
// ANTES (problemático):
function updateField(field, value) {
  state[field] = value;
  if (state.errors[field]) state.errors[field] = null;
  render();  // ← ELIMINAR
}

// DESPUÉS (correcto):
function updateField(field, value) {
  state[field] = value;
  if (state.errors[field]) state.errors[field] = null;
  // No llamar render() aquí - solo actualizar estado
}
```

```javascript
// ELIMINAR de render():
setTimeout(() => {
  const closeDropdown = (e) => { ... };
  document.addEventListener('click', closeDropdown);
}, 0);

// AÑADIR fuera de render, solo una vez:
document.addEventListener('click', (e) => {
  if (!e.target.closest('.nav-btn') && !e.target.closest('.dropdown')) {
    state.showLangDropdown = false;
    updateDropdownUI();
  }
  if (!e.target.closest('.country-btn') && !e.target.closest('.country-dropdown')) {
    state.showCountryDropdown = false;
    updateDropdownUI();
  }
});
```

```javascript
// Simplificar toggles - NO llamar render():
function toggleLangDropdown() {
  state.showLangDropdown = !state.showLangDropdown;
  state.showCountryDropdown = false;
  updateDropdownUI();
}
```
